## Content Conversion Plan

### Export Format Decision

Choose export format for blog posts:
- [ ] **HTML** - Keep original formatting
- [ ] **Markdown** - Clean, readable format
- [ ] **Hybrid** - Markdown with HTML where needed

### File Naming Convention

```
blog/posts/YYYY-MM-DD-post-slug.html
```

Example: `blog/posts/2024-03-15-my-first-post.html`

### Content Processing Steps

1. **Export from WordPress**
   ```bash
   wp export --post_type=post --dir=./export/
   ```

2. **Convert to chosen format**
   - Use WordPress to Markdown converter, or
   - Manual HTML cleanup, or
   - Custom script

3. **Extract metadata**
   ```json
   {
     "id": "post-slug",
     "title": "Post Title",
     "date": "2024-03-15",
     "author": "Author Name",
     "categories": ["category1"],
     "tags": ["tag1", "tag2"],
     "excerpt": "Post excerpt...",
     "image": "assets/images/blog/featured.jpg"
   }
   ```

4. **Download and organize media**
   ```
   assets/images/blog/
   ```
